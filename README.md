# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные

#### type TPayment (способ оплаты)
*Назначение:* описывает возможные способы оплаты заказа.

*Возможные значения:*
```typescript
type TPayment = 'card' | 'cash' | '';
```

#### interface IProduct (товар) 
*Назначение:* описывает товар, который будет использоваться в приложении. Основной интерфейс.

*Структура:* 
```typescript
interface IProduct {
  id: string;           // уникальный идентификатор товара
  description: string;  // подробное описание товара
  image: string;        // путь к изображению для карточки товара
  title: string;        // название товара
  category: string;     // группа, к которой относится товар
  price: number | null; // цена товара
}
```

*Примечание:* поле `price` может иметь значение null для товаров, которые не имеют цены ("бесценные товары")

#### interface IBuyer (покупатель) 
*Назначение:* описывает данные покупателя для оформления заказа. 

*Структура:* 
```typescript
interface IBuyer {
  payment: TPayment;   // способ оплаты
  email: string;       // электронная почта покупателя
  phone: string;       // телефон покупателя
  address: string;     // адрес доставки
}
```

*Примечание:* 
  * Поле `payment` имеет тип `TPayment`, который определен отдельно. Может иметь значения 'card' (оплата онлайн), 'cash' (оплата при получении) или '' для невыбранного способа.
  * Все поля обязательны для заполнения при оформлении заказа.

#### interface IValidationErrors (ошибки валидации) 
*Назначение:* описывает объект с ошибками валидации. Значение - строка с ошибкой, если есть.

*Структура:* 
```typescript
interface IValidationErrors {
  payment?: string;     // ошибка для поля: способ оплаты
  email?: string;       // ошибка для поля: электронная почта покупателя
  phone?: string;       // ошибка для поля: телефон покупателя
  address?: string;     // ошибка для поля: адрес доставки
}
```

#### interface IProductResponse (ответ сервера с эндпойнта /product/)
*Назначение:* описывает объект ответа сервера. 

*Структура:* 
```typescript
interface IProductResponse {
    total: number;       // количество товаров в каталоге
    items: IProduct[];     // массив товаров в каталоге
}
```

#### interface IOrder (ответ сервера с эндпойнта /order/)
*Назначение:* описывает объект ответа сервера. Наследует свойства типа IBuyer.

*Структура:* 
```typescript
interface IOrder extends IBuyer {
    total: number;       // общая сумма заказа
    items: string[];     // массив идентификаторов заказанных товаров
}
```

#### interface IOrderSuccess (успешный ответ сервера с эндпойнта /order/)
*Назначение:* описывает объект ответа сервера. 

*Структура:* 
```typescript
interface IOrderSuccess {
    id: string;          // id заказа
    total: number;       // общая стоимость заказа от сервера
}
```

#### interface IOrderError (ответ сервера с ошибкой с эндпойнта /order/)
*Назначение:* описывает объект ответа сервера. 

*Структура:* 
```typescript
interface IOrderError {
    error: string;       // ошибка
}
```

### Модели данных

#### Класс ProductCatalog (каталог товаров)
Используется для хранения всех товаров, которые можно купить в приложении. Обрабатывает выбор товара для детального просмотра.

Конструктор:  
`constructor()` - конструктор класса не принимает параметров. Инициализирует пустой список товаров. Устанавливает значение выбранной карточки в `null` - не выбрано.

Поля класса:  
`products: IProduct[]` - массив всех товаров.
`selectedProduct: IProduct | null` - товар, выбранный для подробного отображения в модальном окне. Товар может быть не выбран.

Методы класса:  
`getItems(): IProduct[]` - получение массива всех товаров в каталоге. 
`setItems(items: IProduct[]): void` - сохранение массива товаров полученного в параметрах метода.
`getProductById(id: string): IProduct | undefined` - получение товара по его идентификатору. Возвращает `undefined`, если товар не найден.
`selectProduct(product: IProduct): void` - сохранение товара для подробного отображения в модальном окне. Вызывается при клике на карточку товара.
`getSelectedProduct(): IProduct | null` - получение товара для подробного отображения в модальном окне или `null`, если товар не выбран. Используется модальным окном для получения данных.

#### Класс ShoppingCart (корзина с товарами)
Используется для хранения и управления товарами, которые пользователь выбрал для покупки.

Конструктор: 
`constructor()` - конструктор класса не принимает параметров. Инициализирует пустой список товаров в корзине. 

Поля класса:  
`shoppingProducts: IProduct[]` - массив товаров, выбранных покупателем для покупки.

Методы класса:  
`getShoppingProducts(): IProduct[]` - получение массива товаров, которые находятся в корзине.
`addProductToCart(product: IProduct): void` - добавление товара, который был получен в параметре, в массив корзины.
`deleteProductFromCart(id: string): void` - удаление товара, id  которого получено в параметре из массива корзины.
`clearShoppingCart(): void` - очистка корзины.
`getCostShoppingProducts(): number` - получение стоимости всех товаров в корзине.
`getItemsCount(): number` - получение количества товаров в корзине.
`isProductInCart(id: string): boolean` - проверка наличия товара в корзине по его id, полученного в параметрe метода.

#### Класс Customer (покупатель)
Используется для хранения данных покупателя, которые тот должен указать при оформлении заказа и проверки данных при оформлении заказа.

Конструктор: 
`constructor()` - конструктор класса не принимает параметров. Инициализирует поля пустыми значениями.

Поля класса:  
`payment: TPayment` - вид оплаты.
`email: string` - email покупателя.  
`phone: string` - телефон покупателя.   
`address: string` - адреc покупателя.

Методы класса:
`setCustomerData(data: Partial<IBuyer>): void` - сохранение данных в модели. Можно сохранить только одно значение, например, только адрес или только телефон, не удалив при этом значения других полей, которые уже могут храниться в классе.
`getCustomerData(): IBuyer` - получение всех данных покупателя.
`clearCustomerData(): void` - очистка данных покупателя.
`validationData(): ValidationErrors` - валидация данных. Поле является валидным, если оно не пустое. Метод валидации дает возможность определить не только валидность каждого отдельного поля, но и предоставлять информацию об ошибке, связанной с проверкой конкретного значения.

### Слой коммуникации

#### Класс WebLarekApi 
Отвечает за получение данных с сервера и отправку данных на сервер. При работе использует функциональность класса Api и дополняет ее, организуя работу с эндпойнтами `/product/` и `/order/` для `https://larek-api.nomoreparties.co`.

Конструктор: 
`constructor(api: IApi)` - в конструктор передается объект, соответствующий интерфейсу IApi.

Поля класса:  
`api: IApi` - объект, соответствующий интерфейсу IApi.

Методы класса: 
`getProducts(): Promise<IProduct[]>` - выполняет GET запрос на эндпоинт `/product/` и возвращает массив товаров.
`postOrder(data: IOrder): Promise<IResultOrder>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на эндпоинт `/order/` и возвращает ответ. 

*Примечание:* 
  * Параметр `data` имеет тип `IOrder`, который определен отдельно. Тип `IResultOrder` также определен отдельно и может быть одним из двух типов: `IOrderSuccess` или `IOrderError`.

#### Класс Header (компонент Шапка)
Является дочерним классом базового класса `Component<IHeader>`.
Используется для хранения ссылки на элемент-счетчик и элемент-кнопки корзины. С помощью сеттера и унаследованного от базового класса метода `render` обновляет содержимое разметки.

Конструктор:  
`constructor(protected events: IEvents, container: HTMLElement)` - принимает экземпляр брокера событий и ссылку на DOM элемент за отображение, которого он отвечает. В конструкторе происходит вызов `super(container)` для инициализации родительского класса, поиск в корневом элементе этого компонента `(this.container)` элемента-счетчика и элемента-кнопки корзины и сохранение в поля класса. И установка слушателя на событие `click` кнопки открытия корзины для оповещения о событии `basket:open`.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента, унаследовано от родительского класса. Доступно после вызова `super(container)` в конструкторе.
`counterElement: HTMLElement` - поле для хранения элемента-счетчика, найденного в контейнере.
`basketButton: HTMLButtonElement` - поле для хранения элемента-кнопки корзины, найденного в контейнере.
`events: IEvents` - брокер событий для взаимодействия с Презентером.

Методы класса:  
`set counter(value: number)` - Сеттер для поля-счетчика, обновляет текстовое содержимое в разметке.
`render(data?: Partial<IHeader>): HTMLElement` - Метод класса, унаследован от базового класса Component. Он принимает данные, соответствующие интерфейсу `IHeader`, копирует эти данные в поля экземпляра класса, срабатываeт сеттер и таким образом обновляется DOM-элемент. Возвращает ссылку на DOM-элемент.

#### interface IOrderSuccess
*Назначение:* описывает интерфейс объекта успешного заказа. 

*Структура:* 
```typescript
interface IOrderSuccess {
  successCost: number;          // стоимость успешного заказа 
}
```

#### Класс OrderSuccess (компонент Успешный заказ)
Является дочерним классом базового класса `Component<IOrderSuccess>`.
Используется для хранения ссылки на элемент для вывода информации об успешном заказе и элемент-кнопки "За новыми покупками". С помощью сеттера и унаследованного от базового класса метода `render` обновляет содержимое разметки.

Конструктор:  
`constructor(protected events: IEvents, container: HTMLElement)` - принимает экземпляр брокера событий и ссылку на DOM элемент за отображение, которого он отвечает. В конструкторе происходит вызов `super(container)` для инициализации родительского класса, поиск в корневом элементе этого компонента `(this.container)` элемента для вывода информации об успешном заказе и элемента-кнопки "За новыми покупками" и сохранение в поля класса. И установка слушателя на событие `click` кнопки для оповещения о событии `order:success`.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента, унаследовано от родительского класса. Доступно после вызова `super(container)` в конструкторе.
`orderSuccessElement: HTMLElement` - поле для хранения элемента для вывода информации об успешном заказе.
`successCloseButton: HTMLButtonElement` - поле для хранения элемента-кнопки "За новыми покупками", найденного в контейнере.
`events: IEvents` - брокер событий для взаимодействия с Презентером.

Методы класса:  
`set successCost(value: number)` - Сеттер, обновляет текстовое содержимое элемента, отображая итоговую сумму успешного заказа.
`render(data?: Partial<IOrderSuccess>): HTMLElement` - Метод класса, унаследован от базового класса Component. Он принимает данные, соответствующие интерфейсу `IOrderSuccess`, копирует эти данные в поля экземпляра класса, срабатываeт сеттер и таким образом обновляется DOM-элемент. Возвращает ссылку на DOM-элемент.
